Securing Iframes

In the next Compute Studio release, we are adding the capability to publish dynamic visualizations. These visualizations can be viewed on the C/S website or be embeded on another website. However, someone has to pay for the compute that is used to run these visualizations if they are not sponsored. So, the current thinking is that the site that is embedding the viz will also pay for the compute. However, we need to make sure that other people aren't taking advantage of their iframe link and using it on their website.

This is where the Content-Security-Policy frame-ancestors directive is useful. This lets you specify which sites are allowed to embed the viz. It makes sure that the parent of the Iframe has a URL that matches the URL set in the response header.

Hopefull this renders with the correctly once the CSP in place and does not render otherwise.
